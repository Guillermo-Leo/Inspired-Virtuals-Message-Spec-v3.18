League Update

The League Update message is, optionally, sent after the Result message when a Matchday League event has just finished playing. Provides information about the current standings of the league.

Each "Message" element has an "event" sub-element

Each "event" element has the following attributes:

  • EventId – an integer that uniquely identifies the event.
  • EventGUID – a GUID that uniquely identifies the event.
  • EventType - an integer that uniquely identifies the event type.
  • EventTime - the time the event is due to start.
  • MatchdayID - ID of the matchday
  • LeagueName - Name of the league
  • LeagueDay - The league day number. The first round of matches is league day 1, the second round of matches league day 2, etc.
  • xsi:type - A name that corresponds to the XSD that this message will conform to.

Within the event element there are a number of Team child elements. The Team element contains all of the league table information for each team, in position order, they contain the following attributes:

  • Position. The position of this team in the league table.
  • PositionVariation. The direction of movement for this team in the table after the most recent match. Values are “UP”, “DOWN” or “EQUAL”.
  • TeamName. The name of the team at this position.
  • MatchesPlayed. The total number of matches this team has played.
  • MatchesWon. The total number of matches this team has won.
  • MatchesDrawn. The total number of matches this team has drawn.
  • MatchesLost. The total number of matches this team has lost.
  • GoalsScored. The total number of goals scored by this team.
  • GoalsAgainst. The total number of goals scored against this team.
  • Points. The total number of points this team has in the league.
  • Trend. 6 character string representing the last 6 games played, the leftmost string being the most recent "V" is a win for the team "N" is a draw, "P" is a loss and "-" is a game that hasn't happened (used when there have been less than 6 games)
