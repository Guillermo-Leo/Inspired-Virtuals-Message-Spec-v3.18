Event Card

The event card message is how new events are entered into the BMS. It is usually sent on controller startup (providing events aren't already scheduled) and once per day at a fixed time, although there are configuration options that allow the controller to send new events throughout the day.

An event send may be split into batches of a configurable number of events. By default events are sent in batches of 10 but this can be increased or decreased if needed.

Within the message element there is an "events" with a NumEvents="X" attribute, where X determines how many events are contained in the message. The "events" element has an "event" sub-element for each event.

Each "event" element has attributes that are common to all event types. These common attributes are:

  • EventId – an integer that uniquely identifies the event.
  • EventGUID – a GUID that uniquely identifies the event.
  • EventType - an integer that uniquely identifies the event type.
  • EventTime - the time the event is due to start.
  • OddsType – the odds/price format being used. This is either ‘Decimal’, ‘Fractional’ or ‘American’.
  • xsi:type - A name that corresponds to the XSD that this message will conform to.

There may also be event specific attributes or elements. These are described in each event type's event specific page

Each Event element contains a number of Market sub-elements containing the following attributes:

  • Type – bet type name
  • Margin – the Margin value associated with this bet market.
  • BetCode – the numeric code uniquely assigned to this particular type of bet.

With a few expections each Market element contains a number of Selection sub-elements, each representing a possible outcome for this market and containing the following attributes:

  • Num - outcome index. This will correspond to an index in the result message if this outcome is a winning outcome.
  • Price - outcome price. Fractional, Decimal or American depending on OddsType.
  • Outcome - outcome name – these differ per market and are explained in each event type's section of this document.
  • RTP - The percentage return to player that can theoretically be expected from this outcome based on the price and probability. RTPs are acheived by averaging over a large number of events and will not be acheived on a per event basis. This will not necessarily correspond to the margin in the market element or the other outcomes in the same market due to price rounding and price substitution (where an identical outcome appears in two markets it must always appear with the same price).

Any markets that differ from the standard format are described in each event type's event specific page.

Each Event element may contain a number of PartialMarket sub-elements containing the following attributes:

  • Type – bet type name
  • Margin – the Margin value associated with this bet market.
  • BetCode – the numeric code uniquely assigned to this particular type of bet.

Each PartialMarket element contains a number of Selection sub-elements, each representing a possible outcome for this market and containing the following attributes:

  • Num - outcome index. This will correspond to an index in the result message.
  • Price - outcome price. Fractional, Decimal or American depending on OddsType.
  • Outcome - outcome name – these differ per market and are explained in each event type's section of this document.
  • RTP - The percentage return to player that can theoretically be expected from this outcome based on the price and probability. RTPs are acheived by averaging over a large number of events and will not be acheived on a per event basis. This will not necessarily correspond to the margin in the market element or the other outcomes in the same market due to price rounding and price substitution (where an identical outcome appears in two markets it must always appear with the same price).

Any partial markets that differ from the standard format are described in each event type's event specific page.
