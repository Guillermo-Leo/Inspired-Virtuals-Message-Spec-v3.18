Introduction

Overview

This document describes the Virtual events message exchange between the Inspired Virtual Controller application and the licensee’s back-end systems (The BMS or Bet Management System). Using this message exchange, events can be input into the client's backend systems where they can be made into betting opportunities. Events can also be closed for betting, voided and resulted. Communications are two-way, synchronous and always initiated by the Controller. There are sample XML files that accompany this document.

This document should be read alongside the sample XML and the Integration Checklist.

Message Types

There are 6 possible types of message sent to downstream systems these messages will require a response message being returned by the licensee’s systems.

  • Event Card - contains the date/time of the event, the names and prices of the racers, etc. - sent in advance of the event, in a batch or as individual messages, as part of an overnight process.
  • No More Bets - indicating that betting must be closed for an event - sent just before the event starts. [Note: No ‘Off’ message is issued. No More Bets should be used to close bets in the licensee’s back-end systems as it is sent approximately 5 seconds before the ‘Off’.]
  • Result - the result of the event and an indication that bets can be settled – normally sent just after the event finishes.
  • Void - indicates that the event should be voided, and any bets must be reimbursed – this could be sent either if the event was removed from the schedule before it was run, or if an event failed to finish due to a hardware or similar problem.
  • League Update – sent after the Result message when a Matchday League event has just finished playing. Provides information about the current standings of the league.
  • Goal Timings – Optionally sent after the No More Bets message when a Matchday event is about to start. Provides timings for when each of the goals will be scored in the Matchday.

Communication Protocols

Messages can be sent via HTTP/HTTPS POST. The Content-type is "text/xml".
